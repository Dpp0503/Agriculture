# 🌾 Rice Yield Prediction — End-to-End Deep Learning Pipeline

A complete pipeline for **Kharif rice yield prediction** using real meteorological data, synthetic data augmentation (VAE), and sequential deep learning models (LSTM, 1D-CNN, Transformer).

## ⚙️ How Each Script Works

---

### 1. `generate_best_rice_synthetic.py` — VAE-Based Synthetic Data Generator

**Purpose:** Generate realistic synthetic seasonal rice data using a Variational Autoencoder (VAE) and a Random Forest yield predictor.

**Data flow:**
1. Loads `rice.csv` (real weekly data).
2. Aggregates weekly rows → one seasonal row per `(District, Season, Year, Crop)` with features: `Mean_Temp`, `Max_Temp`, `Total_Rain`, `Rain_Days`, `Temp_Variability`, `Mean_Humidity`, `Total_GDD`, `Mean_Solar`, `Early_Rain_30pct`, `Late_Rain_30pct`, `Duration_Weeks`.
3. Trains a **VAE** (encoder: 2-layer MLP → latent dim 4; decoder: mirrors encoder) on scaled weather features.
4. Trains a **Random Forest** on real features → real yield to learn the weather-to-yield mapping.
5. Samples 2000 novel points from the latent space, decodes them, clips to physical bounds (humidity ≤ 100, all values ≥ 0).
6. Predicts yield using RF + adds Gaussian noise sampled from the RF's residual distribution.
7. **Novelty filter:** computes distance from each synthetic point to its closest real neighbor (NearestNeighbors). Drops points with distance ≤ 0.1 ("too close" / memorisation risk).
8. Saves up to 1000 novel samples.

**Key parameters:**
- `latent_dim = 4`, VAE loss = MSE + 1.2 × KLD (β-VAE style)
- `n_estimators = 200` RF, `oob_score = True`
- `dist_threshold = 0.1`

**Output:** `best_synthetic_rice.csv` (~1000 rows × seasonal features), `dcr_plot.png`

---

### 2. `generate_weekly_from_seasonal.py` — Weekly Pattern Reconstruction

**Purpose:** Upgrade synthetic seasonal totals back to a full weekly time-series by borrowing temporal patterns from the most similar real season.

**Data flow:**
1. Loads `rice.csv` (real weekly) and `best_synthetic_rice.csv` (synthetic seasonal).
2. Aggregates real weekly data to seasonal summaries (`Mean_Temp`, `Total_Rain`, `Mean_Humidity`) for pattern matching.
3. Fits `NearestNeighbors` on real seasonal summaries (StandardScaler applied).
4. For each synthetic season, finds the nearest real season.
5. Copies that real season's week-by-week pattern, then **rescales** it:
   - Precipitation scaled by `syn_total_rain / real_total_rain`
   - Temperature (`temp`, `tempmax`, `tempmin`) scaled by `syn_mean_temp / real_mean_temp`
   - Humidity scaled by `syn_mean_hum / real_mean_hum` (clipped 0–100)
6. Stamps `Synthetic_ID`, `Is_Synthetic=1`, and the synthetic yield label onto each week.
7. Generates a validation plot of 3 random synthetic rain curves.

**Output:** `synthetic_weekly_rice.csv` (one row per week per synthetic season), `weekly_pattern_check.png`

---

### 3. `advanced_feature_engineering.py` — Stage-Wise Feature Engineering

**Purpose:** Convert weekly meteorological data into **agronomically meaningful stage-level features** for analysis and validation.

**Crop stage mapping (Kharif calendar):**

| Stage | SMW Weeks | Calendar |
|---|---|---|
| Germination | 22–23 | Early June |
| Vegetative | 24–30 | Mid June – Late July |
| Reproductive | 31–36 | August – Mid September |
| Ripening | 37–44 | Late September – October |

**Data flow:**
1. Loads `rice.csv` and `synthetic_weekly_rice.csv`, combines them.
2. Assigns each weekly row a `Stage` label based on the `week` (SMW) number.
3. Groups by season ID and computes per-stage features:
   - **Rain:** `Rainfall_Total`, `Rainy_Weeks` (weeks > 5 mm), `Max_Weekly_Rain`
   - **Temperature:** `Mean_Temp`, `Heat_Stress_Weeks` (tempmax > 35°C), `Night_Stress_Weeks` (tempmin > 25°C)
   - **Other:** `Mean_Humidity`, `Mean_Solar`
4. Adds **interaction features** for each stage:
   - `{Stage}_Rain_x_Temp` = total rain × mean temp
   - `{Stage}_Heat_x_Humidity` = heat stress weeks × mean humidity
5. Merges with metadata (`District`, `Year`, `Is_Synthetic`) and target (`Yield (Ton./Ha.)`).

**Output:** `advanced_features.csv` — one row per season, ~40+ columns

---

### 4. `check_agronomic_plausibility.py` — Domain Validation

**Purpose:** Verify that the engineered features make agronomic sense before model training.

**Data flow:**
1. Loads `advanced_features.csv`.
2. Computes Pearson correlations between 6 key features and `Yield (Ton./Ha.)`:
   - `Vegetative_Heat_Stress_Weeks` — expected negative correlation
   - `Vegetative_Heat_x_Humidity` — expected negative
   - `Ripening_Rainfall_Total` — expected negative (excess rain at ripening reduces yield)
   - `Ripening_Rainy_Weeks` — expected negative
   - `Reproductive_Mean_Solar` — expected positive (sunlight aids grain filling)
   - `Germination_Mean_Temp` — context dependent
3. Saves correlations to `agronomic_report.txt`.
4. Plots scatter charts: Yield vs Ripening Rainfall and Yield vs Vegetative Heat Stress.

**Output:** `agronomic_report.txt`, `yield_vs_ripening_rain.png`, `yield_vs_heat_stress.png`

---

### 5. `compare_models_comprehensive.py` — Model Benchmarking

**Purpose:** Head-to-head comparison of four deep learning models (RNN, LSTM, CNN, Transformer) on the same train/test split.

**Data flow:**
1. Loads `advanced_features.csv` (tabular features), `rice.csv` (real sequences), `synthetic_weekly_rice.csv` (synthetic sequences).
2. Reconstructs aligned `(District, Year, Is_Synthetic)` keys to match tabular rows to their weekly sequences.
3. Pads/truncates each season's weekly sequence to `seq_len = 20` weeks × 6 features (`precip`, `temp`, `tempmax`, `tempmin`, `humidity`, `solarradiation`).
4. Splits matched data 80/20.
5. Trains and evaluates:
   - **RNN** — `nn.RNN(hidden=64)` → `Linear(1)`, last hidden state
   - **LSTM** — `nn.LSTM(hidden=64)` → `Linear(1)`, last hidden state
   - **1D-CNN** — `Conv1d(64) → Pool → Conv1d(32) → Pool → Linear(1)`
   - **Transformer** — `Linear embedding(32) → 2-layer TransformerEncoder(nhead=4) → Linear(1)`
6. All DL models: `StandardScaler` per feature, `Adam(lr=0.001)`, `MSELoss`, 50 epochs.
7. Finds the best DL model by R² and saves a scatter plot.

**Output:** `comprehensive_model_comparison.png`, `best_dl_vs_actual.png`, `comprehensive_report.txt`, `deep_learning_results.csv`

---

### 6. `train_final_model.py` — Final Model Training & Selection

**Purpose:** Production-grade neural training pipeline — trains all DL models on real + synthetic data with **sample weighting**, and **saves the single best performer** evaluated strictly on held-out real samples.

**Data flow:**
1. Loads `rice.csv` + `synthetic_weekly_rice.csv`.
2. Computes derived features: `GDD = ((tempmax + tempmin) / 2 − 10).clip(0)` and `DTR = tempmax − tempmin`.
3. Prepares **sequential data** (DL input): pads/truncates weekly rows to `SEQ_LEN=26` × 11 features (`temp`, `tempmax`, `tempmin`, `precip`, `humidity`, `solarradiation`, `GDD`, `DTR`, `windspeed`, `cloudcover`, `sealevelpressure`).
4. Splits data 80/20 with **stratified split** (preserves real/synthetic ratio).
5. Target normalization — `StandardScaler` fit **only on training targets** (no leakage).
6. Trains 3 neural models with **real-sample weighting** (10× weight for real data):
   - **LSTM** — 2-layer Bidirectional LSTM (`hidden=64`) with **attention pooling** over all timesteps
   - **1D-CNN** — 3-layer `Conv1d(64→128→128)` + `BatchNorm1d` + residual skip connection + `AdaptiveMaxPool1d(1)`
   - **Transformer** — `Linear(64)` + sinusoidal positional encoding → 3-layer `TransformerEncoder(nhead=4, ff=256)` + `AdaptiveAvgPool1d` + `LayerNorm` + `Dropout`
7. **Training features:**
   - **Huber Loss** (robust to yield outliers)
   - **CosineAnnealingWarmRestarts** LR scheduler
   - **Gaussian noise augmentation** (σ=0.03) on inputs
   - **Padding mask** passed to Transformer during training and inference
   - **Early stopping** on real-data validation set (patience=20)
   - **Two-phase training:** Phase 1 on all data (weighted) → Phase 2 fine-tune on real only (lr=1e-4, patience=10)
   - **Gradient clipping** (max_norm=1.0) and **weight decay** (1e-4)
8. **Ensemble:** averages predictions from the top-2 models.
9. Selects the model with the highest R² on **real-only** held-out test data.
10. Saves the winner to `final_best_model.pth`.

**Output:** `final_best_model.pth`, `advanced_yield_prediction.png`, `deep_learning_results.csv`

## 📊 Key Design Decisions

| Decision | Rationale |
|---|---|
| VAE for synthetic generation | Learns the underlying distribution of weather patterns; generates diverse, novel samples |
| Nearest-neighbor pattern borrowing | Preserves realistic temporal structure in weekly sequences without a complex time-series VAE |
| Novelty filtering (distance > 0.1) | Prevents synthetic data from being near-copies of real data (privacy + augmentation quality) |
| Real-sample weighting (10×) | With 115 real vs 1000 synthetic samples, weighting ensures the model prioritises real patterns |
| Stratified train/test split | Guarantees proportional real/synthetic representation in both sets |
| Target normalization (train-only fit) | Prevents data leakage from test targets; stabilises gradient magnitudes |
| Padding mask for Transformer | Prevents zero-padded positions from corrupting self-attention weights |
| Huber Loss | More robust to yield outliers than MSE |
| Evaluation on real held-out data only | Ensures the reported metric reflects real-world performance, not synthetic data fit |
| Two-phase training + fine-tuning | Pre-train on all data for pattern learning, then fine-tune on real data for domain adaptation |
